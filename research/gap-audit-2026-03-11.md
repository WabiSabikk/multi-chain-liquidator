## Detailed Gap Analysis

### GAP 1: Zero Successful Liquidations (CRITICAL)

**Fact:** 7,129 liquidations across 4 protocols, 0 by our bot.

**Root causes (already identified):**
1. Wrong DEX factory (FIXED in v0.8.0 — alt router)
2. Wrong fee tiers (FIXED — kHYPE=100, wHYPE/USDC=500)
3. Speculative mode gas drain (FIXED — disabled)
4. Mantle RPC offline (FIXED — switched to dRPC)

**Remaining blockers:**
- Detection speed (see GAP 3)
- Calm market — no new liquidatable positions since fixes
- Need to validate during next crash

### GAP 2: Flash Loan Cost (P0 — IMMEDIATE FIX)

**Current:** Our HyperEVM contract uses HyperLend flash loans (4 bps fee).

**Competitors:**
- `0x1ea48b` (Top2, 50 liqs) → Morpho FL = **0 bps**
- `0xa7d048` (81 liqs) → Morpho FL = **0 bps**

**Fix:** Modify `FlashLoanLiquidator` contract to:
1. Request flash loan from Morpho (`0x68e37dE8...morpho.flashLoan()`)
2. Execute liquidation on HyperLend
3. Swap collateral
4. Repay Morpho (principal only, no fee)

**Estimated savings:** On avg liq of $200 debt → $0.08 saved. On $10K liq → $4 saved.

### GAP 3: Detection Speed (P1 — HIGHEST IMPACT)

**The #1 competitive advantage is detection speed, not gas.**

**Evidence:**
- Pearson correlation gasPrice vs txIndex: **-0.12** (weak) → NOT a gas war
- Most liqs have **0 direct competition** — first detector wins
- Top1 (`0x8715ed`): 0.12 gwei gas, txIndex=1 → FAST, not rich
- `0xdd8692`: 3,145 gwei gas, txIndex=0 → compensates slowness with gas

**Our current detection:**
- Poll every ~1.5s (getUserAccountData via multicall)
- React AFTER HF < 1.0 confirmed
- ~1-3 block delay from HF crossing to TX submission

**Competitor likely approach:**
- Monitor oracle price feeds directly
- Pre-compute HF locally when oracle updates
- Have TX ready before HF officially crosses 1.0
- Some may use precompile 0x0807 (HyperCore oracle)

**Fix (Phase 10 — Oracle Prediction):**
1. **Precompile 0x0807** — read HyperCore oracle, check for drift vs HyperLend oracle
2. **CEX WebSocket feeds** — Binance/OKX/Bybit weighted median (1-3s pre-block)
3. **Pre-signed TX cache** — have TX ready, only send raw TX when triggered

### GAP 4: No Batch Liquidation (P2)

**Evidence:** Top1 (`0x5d5ecb`) liquidated 3 users in 1 block (27043201) during Feb 13-14 crash.

**Impact:** During crashes, 10-50+ positions go underwater simultaneously. Without batching:
- We process one at a time (sequential scan → execute)
- Top1 captures 3+ in a single TX
- By the time we try #2, Top1 already got it

**Fix:**
1. After scan finds N liquidatable positions, sort by profit descending
2. Batch top-K into single multicall TX
3. Use Morpho FL for total debt amount, then liquidate all K positions

### GAP 5: No HypurrFi Contract (P3)

**Status:** Monitor detects 205 at-risk positions on HypurrFi. Cannot execute (no contract).

**Market:** 211+ liqs (partial scan), 12 bots. `0xdd8692` active here (6.2%).

**Fix:** Deploy HypurrFi-specific contract:

### GAP 6: Router Overhead (P3)
